Ruby Class Methods

February 4, 2015

This week's post is about class methods in Ruby. Methods in general are a primary building block of object-oriented programming, as they are the way that messages are passed between objects. They are how objects interact with one another. This post focuses on class methods, which work a bit differently than you might be used to seeing. Let's take a look:

What are class methods?

The long and short of it is that class methods are methods that act on the class itself and instance methods are methods that can be called on or within an instance of that class. Class methods will only exist inside of the class that defined them. But what does that really mean?

When to use them

When to use class methods is very much up for debate. Some people completely advocate against using them at all, unless absolutely necessary. Some people goes as far as to claim their hatred for class methods while others merely find them annoying.

Eventually we'll get to the point, if you're not there already, where you develop your own taste for such matters. At this point in my education as a novice developer, the discussions swirling around class methods are way deeper than I am ready for and fly way above my head. It's worth it to be aware of the hot topics though.

My general advice would be to use class methods when you need to perform functions only on the class itself on not on its instances. Looking back to the earlier Cupcake example, while each type of cupcake that you can make and instantiate the class with will have its own unique recipe, the class method recipe_format doesn't actually speak to that. It is a wider function that belongs to Cupcake in general. It belongs to the "thing" that makes kinds of cupcakes, not the kinds of cupcakes themselves. The last step of the recipe_format calls for telling the user how to decorate cupcakes which makes it unique to the Cupcake class rather than applicable to any sibling or parent food class.
