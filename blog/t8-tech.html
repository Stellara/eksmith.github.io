A brief intro to regex

Regex, or regular expressions, are expressions to define a pattern of characters to match in a string. They are universal and portable. Once you know their syntax, you can use them in any language or even just in text editor. They are definitely worth learning.

What can you do with regex?

If you're doing any work with strings at all, using regular expressions provide immense flexibility in how to process them. Some concrete examples of this are in web development, where regular expressions are used to detect if the email, phone number and address fields of a form contain valid input. Regular expressions can also be used to "clean" input, or to make sure a user is only inputting certain characters. The title of this post is also a very basic example of "sanitizing input".

How to use them

On a very basic level, regex consist of some variation of this pattern:

[set of characters]{number of characters to match}

The specifics of how to implement a regex change from language to language, but the syntax will mostly be the same. Some languages such as Java and Ruby require you to create regex objects and with some such JavaScript, you can just work on the string directly (JavaScript will turn the string into a regex object internally). If you're just doing work in your text editor on strings, then you can simply use your editor's find and replace feature, check regex or grep in its options (or settings, etc, this will depend on what you're using) and then enter the regex in find or replace, depending on what you're after.

One of the very most basic regex you can make is simply a literal: /thingtofind/. With this simple pattern, you are looking for exactly "thingtofind". You can do this with whole strings or single characters. Most of the time though, you'll want to use the special characters that are not literals and that denote special conditions. This example is written in JavaScript and is a simple "input sanitation". You can see the starting string is full of crazy symbols. The regex following the replace simply is taking the criteria [^A-Za-z\d_-] and searching for everything that matches it and then replacing that with, well...nothing, as denoted by "". The carrot ^ means "not". A-Za-z simply means any characters uppercase or lowercase a through z, \d means any digit and the underscore and dash are just literally underscore and dash. In plain English, the regex is matching anything in the target string that is not a letter, number, underscore, or dash, and replacing it with nothing, effectively deleting it.

Regex Has Its Limits Though

While regex is super flexible and useful, it can't be applied to everything. Using it to validate an e-mail address or street address is such a complex use case that many people argue against trying, many people state that it just can't be done, and others argue for ditching a regex approach altogether for those cases and using a parser. There is an interesting thread on the topic here, at StackOverflow. Jeff Atwood of Coding Horror fame also has a great article here on regex use vs abuse.

Some Resources to Learn Regex

If you're a beginner like me and you'd like to learn regex there are myriad places to do so. The current resource I am using is Zed Shaw's Learn Regex the Hard Way. His whole Learn x the Hard Way series is fantastic and interactive. You're not just passively reading articles, but you're running the code yourself from your command line. It's pretty legit.

If you are a Rubyist, Rubular is an excellent regex editor, to test your expressions out in.
